# Lab 2: Use Case Diagrams

## Overview
In this lab, two use case diagrams are presented to show how users interact with different systems.

---

## 1. ATM System

This diagram represents how an ATM works.

### Actors
- Customer
- Bank Staff

### Main Functions
- Withdraw Cash
- Check Balance
- Deposit Money
- Authenticate User
- Print Receipt
- Refill ATM Cash

All the main actions done by the customer require authentication. The bank staff is responsible for maintaining the ATM and refilling it with cash.

---

## 2. Medical Clinic System

This diagram shows a system for managing medical appointments.

### Actors
- Patient
- Doctor

### Main Functions
- Log In
- Schedule Appointment
- Cancel Appointment
- Reschedule Appointment
- View Medical Records

Users need to log in before using the system. They can manage appointments and check medical records.

---

## Conclusion

These diagrams help to understand how users interact with the system and what actions they can perform.
